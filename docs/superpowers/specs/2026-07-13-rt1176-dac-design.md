# RT1176 12-bit DAC bring-up — design

Date: 2026-07-13
Status: approved (autonomous session; user brief pre-authorized workflow)
Path decision: **real DAC12** (not MQS) — SDK `dac12/basic` readme for evkbmimxrt1170
confirms DAC0 output = **TP18**, probe-accessible on this board.

## Goal

A true analog voltage output on the RT1176: write a 12-bit code (0–4095) and the
DAC pin (TP18) settles to the corresponding fraction of VREFH. Success:

1. QEMU gate green (architectural observables + model conversion trace).
2. HW: DC level tracks the written code (mid-scale 2048 ≈ 0.9 V), 0→4095 sweep is
   linear against ~1.8 V full scale, and a slow ramp + sine LUT produce the matching
   waveform on a scope/DMM/Saleae-analog at TP18.

This is **not** PWM (`analogWrite` = FlexPWM, see `rt1176-pintable-pwm` memory);
it is the genuine analog-out path. There is **no Teensy 4 reference** — the RT1062
has no DAC — so this is a from-SDK bring-up. The API name comes from Teensy 3.x
(Kinetis), which had real DACs.

## Ground truth (SDK + RM Rev.5 + cm7 header)

Sources: `~/Development/mcuxsdk-ws/mcuxsdk/drivers/dac12/fsl_dac12.{h,c}`,
`.../devices/RT/RT1170/periph/PERI_DAC.h`,
`.../devices/RT/RT1170/MIMXRT1176/MIMXRT1176_cm7_COMMON.h` + `_features.h`,
SDK example `driver_examples/dac12/{basic,fifo_interrupt}` (evkbmimxrt1170 cm7),
IMXRT1170RM Rev.5 ch. 89 (DAC), §89.1 chip-specific info.

- Single instance `DAC`, base **0x40064000**, **IRQ 63** (`DAC_IRQn`, cm7 header).
- Clock gate: **kCLOCK_Dac = LPCG 57** → `CCM_LPCG57_DIRECT @ 0x40CC6720`
  (0x40CC6000 + 57*0x20, same stride as every other LPCG in the core).
- Register layout (PERI_DAC.h; **RT1170 DAC has NO ITRM register** —
  `FSL_FEATURE_DAC12_HAS_NO_ITRM_REGISTER = 1`):

  | offset | reg   | access | notes |
  |--------|-------|--------|-------|
  | 0x00   | VERID | RO     | version id |
  | 0x04   | PARAM | RO     | FIFOSZ[2:0]; reset 0b011 → FIFO depth 16 (2^(FIFOSZ+1)) |
  | 0x08   | DATA  | **WO** | DATA0[11:0] — the converted code |
  | 0x0C   | CR    | RW     | flags[4:0] (FULLF,NEMPTF ro; WMF ro; UDFF,OVFF w1c), FULLIE(8), EMPTIE(9), WTMIE(10), SWTRG(12, self-clear), TRGSEL(13), DACRFS(14), DACEN(15), FIFOEN(16), SWMD(17), UVIE(18), FIFORST(21), SWRST(22), DMAEN(23), WML[31:24] |
  | 0x10   | PTR   | RO     | DACWFP[7:0] write ptr, DACRFP[23:16] read ptr |
  | 0x14   | CR2   | RW     | BFEN(0), OEN(1), BFMS(2), BFHS(3), IREF2(4), IREF1(5), IREF(6) |

- **RM chip-specific constraints (§89.1):** the ONLY reference option is **VREFH**
  (ADC_VREFH pin) — **DACRFS must be set to 1** (`kDAC12_ReferenceVoltageSourceAlt2`,
  which is what the EVKB example uses). VREFL = ADC_VREFL. **No HW trigger** on this
  chip (`FSL_FEATURE_DAC12_HAS_HW_TRIGGER = 0`; TRGSEL should select SW trigger).
  **IREF2 (PTAT) is not available** — use IREF (internal, CR2 bit 6, the SDK default)
  or IREF1 (ZTC). A current reference MUST be selected or the DAC silently doesn't
  convert (fsl_dac12.h warns: "DAC would waste current but no function").
- On the EVKB, VREFH is the 1.8 V analog rail → full scale ≈ 1.8 V; code→volts =
  `code/4096 * VREFH`. Reconcile the measured full-scale on HW.
- Init sequence (fsl_dac12.c, non-FIFO config): enable LPCG → `CR = DACRFS |
  TRGSEL(sw)` (FIFO disabled, watermark 0) → `CR2 = IREF | OEN` (internal current
  ref; output buffer **bypassed** = SDK default `enableAnalogBuffer=false`; low
  speed) → `CR |= DACEN`. In non-FIFO mode **every DATA write converts
  immediately** — no trigger needed (`DAC12_SetData` is the whole write path).

## Arduino API (v1)

**`void analogWriteDAC0(uint32_t value);`** — declared in `core_pins.h` next to
`analogWrite`, implemented in new `cores/imxrt1176/dac.c`.

- Teensy 3.x-compatible semantics: `value` is interpreted at the current
  `analogWriteResolution()` (default 8 bits) and scaled to the DAC's native 12 bits
  (`<< (12-res)` / `>> (res-12)`), clamped to full scale.
- Lazy init: first call enables LPCG57 and programs CR/CR2 as above (matches the
  Teensy convention — no `begin()`).
- Alternatives weighed: a `DAC` class (`begin()/write()`) — rejected, invents a new
  surface where a Teensy-precedent one exists; routing `analogWrite(pin)` — rejected,
  the DAC output is TP18, a dedicated analog pad with no IOMUXC mux and no header
  pin number.
- Out of scope for v1 (natural follow-ons): FIFO/DMA waveform streaming, an
  `analogWriteDAC0`-driven AudioStream node, buffered/high-drive mode selection.

`pwm.c`'s file-static resolution (`pwm_res_bits`) needs to be readable from `dac.c`
— expose via a small internal getter (or make the variable non-static with an
`extern` in a private header), matching whichever is least invasive to pwm.c.

## Register definitions (imxrt1176.h + generator)

Add to BOTH `cores/imxrt1176/imxrt1176.h` and `cores/imxrt1176/tools/gen_imxrt1176_h.py`:
`IMXRT_DAC` struct (VERID, PARAM, DATA, CR, PTR, CR2) at 0x40064000, the CR/CR2/PTR/
PARAM bitfield masks from PERI_DAC.h, `CCM_LPCG57_DIRECT`, and `IRQ_DAC = 63`.

## QEMU model — `hw/misc/imxrt_dac.c` (+ `include/hw/misc/imxrt_dac.h`)

Upgrade the existing `create_unimplemented_device` stub (`{ "dac", 0x40064000,
16 KiB }` in `hw/arm/fsl-imxrt1170.c`) to a functional model, same pattern as
`imxrt_lpadc` (the analog-in twin): sysbus device, one MMIO region, one IRQ,
instantiated in the SoC init with **IRQ 63** wired; meson/Kconfig registration
mirroring `CONFIG_IMXRT_LPADC`.

Behavior modelled (faithful to RM ch. 89):
- `VERID`/`PARAM` read as RM reset values (PARAM.FIFOSZ = 0b011 → depth 16).
- `DATA` write-only; DATA0[11:0] masked. FIFO disabled → immediate "conversion"
  (store as current output code). FIFO enabled → push at write pointer, DACWFP
  advances (wraps at depth), FULLF/OVFF per RM.
- `CR`: DACEN gates conversion; SWTRG (when TRGSEL=sw, FIFO on) pops at read
  pointer → conversion, DACRFP advances, NEMPTF/UDFF per RM; WMF when remaining
  data < WML; FIFORST resets pointers; SWRST resets the block; OVFF/UDFF are W1C;
  interrupt line = (WTMIE & WMF) | (EMPTIE & NEMPTF) | (FULLIE & FULLF) |
  (UVIE & (OVFF|UDFF)).
- `PTR` read-only pointer readback.
- Every conversion emits `qemu_log("imxrt_dac: convert 0x%03x ...")` → lands in the
  runner's `-D` .dbg file for the gate to grep (the "analog" observable in QEMU).
- Not modelled: analog electrical behavior, CR2 effects beyond storage, DMA (v1).

## Gate — `evkb/dac_test/` (sketch + `run_qemu_dac.sh` via qrun)

Copy `analog_test` as the template (CMakeLists/toolchain/runner). **New core .c file
⇒ from-scratch cmake reconfigure** (`rm -rf build && cmake -B build`) — the
file(GLOB) trap.

Anti-circularity rule: `DATA` is write-only on silicon, so the sketch asserts only
**architecturally readable state that behaves identically on HW**:

1. `analogWriteDAC0()` path: call at default 8-bit res (128 → 0x800), at
   `analogWriteResolution(12)` (2048 → 0x800; 4095 → 0xFFF), print CR readback —
   expect DACEN|DACRFS|TRGSEL set, FIFOEN clear. Serial `PASS`/`FAIL` lines.
2. Raw FIFO phase (register-level in the sketch): FIFORST → FIFOEN+WML=4+SWTRG mode,
   write 4 codes → expect `PTR.DACWFP == 4`; SWTRG once → `PTR.DACRFP == 1`;
   drain to nearly-empty → NEMPTF set; watermark IRQ via
   `attachInterruptVector(IRQ_DAC, …)` + NVIC enable → ISR-fired flag printed.
   Proves IRQ 63 wiring end-to-end.
3. Runner additionally greps the QEMU `.dbg` log for the model's conversion traces
   (`convert 0x800`, `convert 0xfff`) — QEMU-only check that resolution scaling
   reached DATA intact.

The same sketch runs unmodified on HW (phases 1–2 must print the same PASSes) —
that is the non-circular half of the triad.

## HW verification (final arbiter)

`evkb/dac_test` flashed via LinkServer (pkill LinkServer/redlinkserv first; pyserial
reader before `LinkServer run`; VCOM @115200). Then a HW sketch mode (same test dir,
compile-time or serial-command switch) that:
- holds mid-scale 2048 → DMM/scope on TP18 ≈ 0.9 V;
- steps 0/1024/2048/3072/4095 (a few seconds each) → linearity vs ~1.8 V full scale;
- outputs a slow ramp and a sine LUT → waveform on scope/Saleae-analog (Logic 2
  automation, `TimedCaptureMode`, analog channel clipped to TP18).
Reconcile measured full scale against VREFH; record results in `HW-RESULTS`-style
notes and a new `rt1176-dac` memory note.

## Risks / notes

- If conversions don't appear on HW: first suspects are the current reference
  (CR2.IREF must be set) and the LPCG gate — both invisible in a QEMU-only pass.
- VREFH exact value is a measurement question (1.8 V nominal); linearity of the
  sweep matters more than the absolute rail.
- DATA is WO: never assert on reading DATA back, in the model or the sketch.
- FlexRAM/OverDrive/startup are already proven; nothing here touches clocks beyond
  LPCG57.
