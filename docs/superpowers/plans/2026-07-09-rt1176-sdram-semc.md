# RT1176 SDRAM (SEMC) Bring-up Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** bring up the EVKB's 64 MB SEMC SDRAM so `EXTMEM` works at `0x80000000` on silicon — faithful QEMU gate (window unmapped until SEMC init completes) + a HW memory test as arbiter.

**Architecture:** Core gets SEMC register defs + ~48 `GPIO_EMC_*` pins + the SEMC clock + a ported `SEMC_ConfigureSDRAM` run early in `startup.c`, plus the `EXTMEM` macro. qemu2 replaces the `semc-ctrl` unimplemented stub with a minimal SEMC model that enables the (initially-disabled) SDRAM window on the MODESET IP command. A `sdram_test` gate runs a memory test.

**Tech Stack:** C (Teensyduino core, RT1176 M7), QEMU, CMake cross-build, LinkServer + LPUART1 VCOM.

**Spec:** `docs/superpowers/specs/2026-07-09-rt1176-sdram-semc-design.md`. Reuses patterns from memory `rt1176-usb-host-hid` (section macros, `.bss.dma` zero-init lesson, gate infra) and the generator/gate discipline.

**SDK references (port from these — do NOT invent):**
- config + init: `mcuxsdk/examples/_boards/evkbmimxrt1170/driver_examples/semc/sdram/{cm7,cm4}/hardware_init.c` (the `sdramconfig` struct — verbatim values in the spec) + `pin_mux.c` (the `IOMUXC_GPIO_EMC_B{1,2}_xx_SEMC_*` list).
- driver: `fsl_semc.c` `SEMC_ConfigureSDRAM` + `SEMC_SendIPCommand`; register layout: CMSIS `PERI_SEMC.h`.

## Task 1: Core SEMC registers + pins + clock + EXTMEM macro

**Files:** `cores/imxrt1176/tools/gen_imxrt1176_h.py` (+ regen `imxrt1176.h`); test: a throwaway compile-check.

- [ ] **Step 1: SEMC register defs.** Add `SEMC_BASE 0x400D4000` + the register set the init uses (from CMSIS `PERI_SEMC.h`): `MCR` (0x00), `IOCR` (0x04), `BMCR0/1` (0x08/0x0C), `BR0..BR8` (0x10..0x30), `INTEN`/`INTR` (0x38/0x3C), `SDRAMCR0..3` (0x40..0x4C), `DBICR0/1`, `IPCR0` (0x90), `IPCR1` (0x94), `IPCR2` (0x98), `IPCMD` (0x9C), `IPTXDAT` (0xA0), `IPRXDAT` (0xB0), `STS0..` (0x3C0-ish). Include the field masks the port needs (`SEMC_BR_VLD_MASK`, `SEMC_BR_MS(n)`, `SEMC_SDRAMCR0_PS/BL/COL/CL`, the `IPCMD` command codes, `IPCMD` done/`IPCR` bits). **Cross-check offsets/bits against `PERI_SEMC.h`.**

- [ ] **Step 2: SEMC pins.** Add the ALT-mux + pad-ctl defs for the SEMC SDRAM pads — `GPIO_EMC_B1_00..B1_26` and `GPIO_EMC_B2_00..B2_20` (data/addr/BA/CS/RAS/CAS/WE/CKE/CLK/DQM/DQS) — from the board `pin_mux.c` (`IOMUXC_GPIO_EMC_B{1,2}_xx_SEMC_*` + the pad config: high-drive, the CLK/CLKX pull-down, others per the SDK). These are numerous; port the list faithfully (one wrong pad = memory corruption).

- [ ] **Step 3: SEMC clock root.** Add the `kCLOCK_Root_Semc` clock-root register + its LPCG. Resolve the exact root index + PLL source/divider from the SDK `clock_config.c` (target ~166–198 MHz); expose a `semc_clock_hz` the init uses for the ns→cycle timing math.

- [ ] **Step 4: `EXTMEM` macro.** Add to the header: `#define EXTMEM __attribute__((section(".externalram"), used))` (parallel to `DMAMEM` at imxrt1176.h ~839). `.externalram` is already in the linker (`imxrt1176.ld` `.bss.extram`) — no linker change.

- [ ] **Step 5: Regenerate + compile-check.** Regen `imxrt1176.h`; confirm `git diff` is only the SEMC/EXTMEM additions. Compile-check a `.c` referencing `SEMC_MCR`, `SEMC_BR0`, `SEMC_SDRAMCR0`, `SEMC_IPCMD`, `EXTMEM int x`.

- [ ] **Step 6: Commit** (teensy-cores): `imxrt1176: add SEMC regs + GPIO_EMC pins + SEMC clock + EXTMEM macro`.

## Task 2: SEMC SDRAM init in the core

**Files:** `cores/imxrt1176/semc.c` (new) + `startup.c` (call it); test: build an EXTMEM-using firmware that links.

- [ ] **Step 1: Port `semc_sdram_init()`** into `semc.c` from `fsl_semc.c::SEMC_ConfigureSDRAM` for the **spec's config table** (64 MB, 32-bit port CS0, burst 8, col 9-bit, CAS 3, the listed timings, 8 auto-refresh). Steps, in order (matches the driver):
  1. ungate the SEMC LPCG + enable the SEMC clock root.
  2. configure the ~48 SEMC pins (Step-2 defs).
  3. `SEMC_MCR` reset/enable; `BR0 = 0x80000000 | SEMC_BR_MS(memsize) | SEMC_BR_VLD` (defines + validates the CS0 region); `SDRAMCR0 = PS(32bit)|BL(8)|COL(9)|CL(3)`; `SDRAMCR1..3` = the timing registers (ns→cycles from `semc_clock_hz`, port the driver's math); `IPCR0` = address, `IPCR1=2`, `IPCR2=0`.
  4. IP-command init sequence via a ported `semc_send_ipcmd`: **Precharge-all → 8× Auto-Refresh → Mode-Set** (mode = burstLen | CAS<<4). Poll `IPCMD`/`INTR` done between commands.
- [ ] **Step 2: Call it early in `startup.c`** — after clocks are up but **before `__libc_init_array`** and before any `.externalram` access (place it near the other early inits; SDRAM must be live before C++ ctors that touch EXTMEM). No `.externalram` zero-init (NOLOAD, per spec).
- [ ] **Step 3: Build-proof** — build a firmware with `EXTMEM uint32_t probe[64];` that writes/reads `probe` in `main`; confirm it links (`probe` at `0x80000000` via `nm`) and compiles clean. Commit (teensy-cores): `imxrt1176: SEMC SDRAM init (SEMC_ConfigureSDRAM port) run early in startup`.

## Task 3: qemu2 minimal SEMC model (faithful window gate)

**Files:** `qemu2/hw/misc/imxrt_semc.c` (new), `hw/misc/meson.build`, `hw/arm/fsl-imxrt1170.c`.

- [ ] **Step 1: Model** a `TYPE_IMXRT_SEMC` sysbus device (16 KB MMIO @ `0x400D4000`) replacing the `semc-ctrl` unimplemented stub. Store the written registers (BR0, SDRAMCR0, IPCR/IPCMD…); reads return them. Model on a simple `hw/misc/` register device.
- [ ] **Step 2: Gate the SDRAM window.** Give the device a link to `s->sdram` (or emit a qemu_irq/gpio the machine wires to enable the region). In the machine, create `s->sdram` **disabled** (`memory_region_set_enabled(&s->sdram, false)` after add_subregion). When the SEMC device sees the **MODESET `IPCMD`** write (command code = SDRAM Mode-Set; O2), call `memory_region_set_enabled(&s->sdram, true)`. (Rationale: `BR.VLD` is set before init; MODESET is the last step → the true "SDRAM ready" point.)
- [ ] **Step 3: Wire into the machine** (`fsl-imxrt1170.c`): instantiate the SEMC device at `0x400D4000` (remove the stub entry), pass it the `sdram` region link, create `sdram` disabled. Build: `ninja -C .../qemu2/build qemu-system-arm`.
- [ ] **Step 4: Smoke test.** Boot a tiny image that reads `0x80000000` before any SEMC write → should read the disabled-region value (0 / abort-handled), and after a MODESET `IPCMD` write → enabled. (Full proof is Task 4.) Commit (qemu2): `imxrt1170: model SEMC enough to gate the SDRAM window on init (was unimplemented stub)`.

## Task 4: sdram_test gate (memory test, faithful)

**Files:** `evkb/sdram_test/{sdram_test.cpp, CMakeLists.txt, toolchain/, .gitignore, run_qemu_sdram.sh, check_sdram.py}`.

- [ ] **Step 1: Firmware** (loop/setup-context Serial1 markers only): `EXTMEM uint32_t buf[SDRAM_TEST_WORDS];` and a memory test:
  - **data-line:** walking 1s and 0s in a single word (`1<<i`, `~(1<<i)`), verify each.
  - **address-line:** write each of a set of power-of-two-strided addresses spanning the 64 MB with a value = its own index; read all back (catches address-line shorts + aliasing).
  - print `SDRAM_INIT`, `SDRAM_DATA=PASS|FAIL`, `SDRAM_ADDR=PASS|FAIL`, `SDRAM_TEST=PASS|FAIL`.
  Keep the test size modest for QEMU speed but stride across the full 64 MB for the address test.
- [ ] **Step 2: CMakeLists + toolchain** — model on an existing gate (e.g. `eeprom_test`); just the core + this firmware (no libraries). `EXTMEM` from the core header.
- [ ] **Step 3: Build.** `rm -rf build && cmake -B build -DCMAKE_TOOLCHAIN_FILE=toolchain/rt1170-evkb.toolchain.cmake && cmake --build build` → `sdram_test.elf`; `nm` shows `buf` at `0x8000_0000`.
- [ ] **Step 4: Runner + checker.** `run_qemu_sdram.sh` (qrun + gate-lib, `-serial file:$OUT`, no `-icount` needed unless timing-sensitive); `check_sdram.py` asserts `SDRAM_INIT` + `SDRAM_DATA=PASS` + `SDRAM_ADDR=PASS` + `SDRAM_TEST=PASS`.
- [ ] **Step 5: Drive to green + prove the gate is faithful.** Run → `SDRAM_TEST=PASS` (proves the SEMC init reached MODESET → window enabled → test passes). Then a **negative check**: a build/variant that skips `semc_sdram_init` (or an early read before init) must **fail** (window disabled) — confirms QEMU isn't false-passing. Fix `imxrt_semc.c`/init as needed; re-run.
- [ ] **Step 6: Commit** (evkb gate; qemu2 fixes if any).

- [ ] If FAIL: a `SDRAM_DATA` fail → a data-line pin/pad; `SDRAM_ADDR` fail → an address-line pin or wrong geometry; total garbage → clock/timing or the init sequence. Record the result for the memory note.
