# RT1176 → RPi 7" Display v1 ("solid color lit") Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Light the Raspberry Pi 7" Touch Display (v1.1) on the MIMXRT1170-EVKB with a single solid color, verified both in a full-transport QEMU model (color visible in the QEMU window + a checksummed pixel-stream oracle) and on real glass.

**Architecture:** SDRAM RGB565 framebuffer (painted by the existing PXP) → LCDIFv2 scanout → MIPI DSI host (DPI video mode) → TC358762 DSI→DPI bridge → panel; ATtiny88 over LPI2C1 powers the panel first. QEMU model and firmware advance in lockstep, one gate token per layer, so a break localizes to the layer that broke.

**Tech stack:** C++ bare-metal (Teensy-derived `cores/imxrt1176`), new MIT library `newdigate/RPiDisplay`, `qemu2` machine model (GPL, firewalled), CMake `teensy-cmake-macros` manifest, QEMU gate via `tools/qrun`, LinkServer HW flash.

**Spec:** [`docs/superpowers/specs/2026-07-23-rt1176-rpi-display-design.md`](../specs/2026-07-23-rt1176-rpi-display-design.md)

## Critical conventions for THIS plan (read before Task 1)

1. **Two-gate rhythm.** Each task adds a gate assertion that FAILS first (red), then the code that makes it pass (green in QEMU via `./run_qemu.sh`), then — at the end — HW. The gate token IS the test.
2. **NEVER fabricate a register value.** Every LCDIFv2/DSI/D-PHY/PLL/TC358762/ATtiny value is transcribed from the **cited local source** named in the task, then *proven* by the gate token or by HW. (This is the exact discipline that caught the fabricated PXP FORMAT enum — a value that "passes QEMU by guessing" and fails on silicon is the failure mode we are engineering against.) If a value cannot be found in the cited source, the task's job is to find and cite it, not invent it.
3. **License firewall.** `qemu2` is GPL — its code NEVER enters a firmware repo. The `newdigate/RPiDisplay` driver is MIT, written as our own Teensy-style code; NXP BSD SDK (`~/Development/mcuxsdk-ws/mcuxsdk/drivers/{lcdifv2,mipi_dsi}`, `devices/RT/RT1170/periph/PERI_*.h`) and RPi Linux drivers are read for **register facts only**, values reproduced, never code-copied. Attribution comment in each driver file.
4. **Self-validating gate.** Expected pixels/checksums are computed in software in the gate, never hard-coded literals. QEMU and HW must agree on `FB_SUM`. `PANEL_SUM` is a QEMU-only oracle (the virtual bridge's received-pixel checksum); on HW that debug register reads a sentinel and the gate prints "verify by eye".
5. **Board flow (HW tasks only).** `pkill LinkServer; pkill redlinkserv` before every flash; start the pyserial reader (`tools/rt1170-console.py`) BEFORE `LinkServer run MIMXRT1176:MIMXRT1170-EVKB <elf>`; VCOM @115200; never `cat` the port. Unplug any USB-OTG2 adapter (A5/AD_08 = OTG2_ID short can kill LPI2C1 — see [[rt1176-a5-ad08-otg2-id-short]]).
6. **Repos & pins.** Firmware driver → new `newdigate/RPiDisplay` (local-first checkout at `~/Development/RPiDisplay`). Core registers → `cores/imxrt1176`. QEMU → `qemu2`. Push and pin-bump only in Task 15, only when the user asks.

- [ ] **Step 2: Write the gate harness with all tokens FAILING.** `rpi_panel_test.cpp` includes the checksum + RGB565↔888 helpers (used by later tasks), calls `Display.begin()` + `fillScreen()`, and emits one token per stage read unconditionally from the `*Ok()` accessors (so a false localizes).
```cpp
#include <Arduino.h>
#include "Display.h"
static const uint16_t COLOR = 0xF800; // solid red (RGB565) — arbitrary, checksum is computed
// FNV-1a over the framebuffer bytes
static uint32_t fnv1a(const uint8_t*p,uint32_t n){uint32_t h=2166136261u;while(n--){h^=*p++;h*=16777619u;}return h;}
// standard bit-replication expansion, shared with PXP fill + the bridge oracle
static inline uint32_t rgb565_to_888(uint16_t c){
  uint32_t r=(c>>11)&0x1F,g=(c>>5)&0x3F,b=c&0x1F;
  r=(r<<3)|(r>>2); g=(g<<2)|(g>>4); b=(b<<3)|(b>>2);
  return (r<<16)|(g<<8)|b;
}
void setup(){
  Serial.begin(115200); while(!Serial && millis()<2000){}
  Serial.println("RPI_PANEL_BEGIN");
  bool ok = Display.begin();
  // stage tokens — emitted unconditionally so the first false pinpoints the broken layer
  Serial.printf("ATTINY_%s\n",  Display.attinyOk()?"OK":"FAIL");
  Serial.printf("PLL_%s\n",     Display.pllOk()?"OK":"FAIL");
  Serial.printf("LCDIFV2_%s\n", Display.lcdifOk()?"OK":"FAIL");
  Serial.printf("DSI_%s\n",     Display.dsiOk()?"OK":"FAIL");
  Serial.printf("TC358762_%s\n",Display.bridgeOk()?"OK":"FAIL");
  if(ok){
    Display.fillScreen(COLOR);
    // FB_SUM: checksum the SDRAM framebuffer we painted (proves PXP.fill)
    uint32_t fb_sum = fnv1a((const uint8_t*)Display.framebuffer(), 800u*480u*2u);
    // expected: whole buffer is COLOR
    uint32_t exp=2166136261u; uint8_t lo=COLOR&0xFF, hi=(COLOR>>8)&0xFF;
    for(uint32_t i=0;i<800u*480u;i++){exp^=lo;exp*=16777619u;exp^=hi;exp*=16777619u;}
    Serial.printf("FB_SUM=%08lX EXP=%08lX %s\n",(unsigned long)fb_sum,(unsigned long)exp, fb_sum==exp?"PASS":"FAIL");
    // PANEL_SUM: QEMU-only oracle — the virtual TC358762's received-pixel checksum (Task 11/13)
    Serial.println("PANEL_SUM_PENDING"); // filled in at Task 13
  }
  Serial.println("RPI_PANEL_END");
}
void loop(){}
```
