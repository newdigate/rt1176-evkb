# RT1176 ST7735 Display (ST7735_t3 blocking-SPI port) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Drive a physical ST7735 128×160 TFT from the MIMXRT1176-EVKB by adding a blocking-SPI `__IMXRT1176__` platform branch to the ST7735_t3 library; success = pixels on glass.

**Architecture:** ST7735_t3 gets a fourth platform branch (alongside KINETISK / `__IMXRT1062__` / `__MKL26Z64__`) modeled on the Teensy-LC pattern: all bus bytes go through the blocking `SPIClass` API (LPSPI1), DC/CS/RST are plain `pinMode`/`digitalWrite` GPIO with a `_dcpinAsserted` state cache. No DMA, no framebuffer, no `IMXRT_LPSPI_t` register access. A QEMU gate proves compile+link+init-runs; real hardware is the arbiter.

**Tech Stack:** newdigate/ST7735_t3 (MIT), newdigate/SPI (`SPIIMXRT1176.h`), imxrt1176 core, teensy-cmake-macros, qrun/gate-lib QEMU harness, LinkServer + pyserial for HW.

**Repos touched:**
- `~/Development/ST7735_t3` — the library (already the `newdigate/ST7735_t3` fork, clean tree, `master`). Port + gate live here.
- `~/Development/rt1170/evkb/cores/imxrt1176` — one new stub header `wiring_private.h` (git repo root is `evkb/cores`).
- `~/Development/SPI` — small cleanup of `tests/st7735_test/st7735_test.cpp` only.
- **Do NOT commit anything in `evkb` itself except the plan/spec docs; the evkb tree has unrelated pre-existing WIP (`wire_*` files) — never stage it.**

**Key facts discovered during design (do not re-derive):**
- The imxrt1176 core has **no** `portOutputRegister`/`portSetRegister`/`digitalPinToBitMask` macros, and `digitalWriteFast()` just calls `digitalWrite()`. All GPIO in the port must be plain `digitalWrite`.
- Our `SPIClass` (`~/Development/SPI/SPIIMXRT1176.h`) exposes the Teensy API: `beginTransaction/endTransaction`, blocking `transfer`/`transfer16`, and stub `pinIsChipSelect()`→false, `setCS()`→0. LPSPI1 only.
- `ENABLE_ST77XX_FRAMEBUFFER` auto-stays-off for `__IMXRT1176__` (the `#ifndef DISABLE_ST77XX_FRAMEBUFFER` block only enables it for MK64/MK66/MK20DX256/`__IMXRT1062__`). No work needed to disable DMA/framebuffer.
- `ST7735_t3.cpp` includes `wiring_private.h`, which the imxrt1176 core lacks (teensy4's is an empty public-domain stub).
- The drawing/font body of `ST7735_t3.cpp` (lines ~960–4079) has zero platform-specific references; all DMA code is inside `ENABLE_ST77XX_FRAMEBUFFER`. `ST7789_t3.cpp`/`ST7796_t3.cpp` have no platform ifdefs of their own.
- `commonInit()` ends with common RST-toggle code using `pinMode`/`digitalWrite` — works as-is with RST=D9.
- `import_arduino_library(NAME ROOT arg…)` globs `ROOT/*.cpp|c|S` plus each `ROOT/<arg>/*` subdir (CONFIGURE_DEPENDS already applied), so `import_arduino_library(ST7735_t3 $ENV{HOME}/Development/ST7735_t3 src)` compiles all of `src/`.
- Pins: SCK=D13, MOSI=D11→SDA, CS=D10, DC=D8, RST=D9. Panel proven by nothing yet (the old WIP sketch was build-only).

```cpp
#if defined(__IMXRT1176__)
  // MIMXRT1176-EVKB (RT1170-EVKB): blocking LPSPI1 via the SPIClass API only.
  // DC (_rs) and CS are plain GPIO — the imxrt1176 core has no port/mask
  // macros, and blocking transfers make digitalWrite between frames safe.
  SPIClass *_pspi = nullptr;
  uint8_t  _cs, _rs, _rst, _sid, _sclk;
  uint8_t  _dcpinAsserted;  // 0=data, 1=command, 2=unknown (forces first write)

  inline void beginSPITransaction() {
    if (_pspi) _pspi->beginTransaction(_spiSettings);
    if (_cs != 0xff) digitalWrite(_cs, LOW);
  }

  inline void endSPITransaction() {
    if (_cs != 0xff) digitalWrite(_cs, HIGH);
    if (_pspi) _pspi->endTransaction();
  }

  void setCommandMode() __attribute__((always_inline)) {
    if (_dcpinAsserted != 1) {
      digitalWrite(_rs, LOW);
      _dcpinAsserted = 1;
    }
  }

  void setDataMode() __attribute__((always_inline)) {
    if (_dcpinAsserted != 0) {
      digitalWrite(_rs, HIGH);
      _dcpinAsserted = 0;
    }
  }
#endif
```
